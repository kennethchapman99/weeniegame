# NPC and Human Framework

Humans should feel like readable systems, not random obstacles.

## Human Types

### Soft-Hearted Human

Used by:
- Dinner Table Begging
- Kitchen Frenzy

Behaviors:
- drops snacks if cute meter fills;
- reacts mildly to mischief;
- can be distracted.

### Strict Human

Used by:
- Dinner Table Begging
- Cleaning Day

Behaviors:
- line-of-sight banishment;
- annoyance meter;
- patrols predictable routes.

### Phone-Absorbed Teenager

Used by:
- Operation Pee Break

Behaviors:
- attention meter;
- responds to leash, shoe, charger, bark rhythm;
- ignores weak signals.

### Cleaner

Used by:
- Cleaning Day Invasion

Behaviors:
- moves chairs;
- vacuums paths;
- puts toys away;
- can be distracted.

### Vet / Groomer

Used by:
- Vet Appointment
- Nail Clip Nightmare

Behaviors:
- hold state;
- exam phases;
- nail tool phases;
- treat distraction windows.

## Shared AI Concepts

- predictable route;
- attention cone;
- annoyance meter;
- distraction window;
- reaction state;
- reset state.

## Design Rule

Humans are not enemies. They are giant confusing forces in the dog world.

Failure should be comedic, not mean.
